                        Sistema de livro de receitas 
Funcionamento do projeto
-O sistema terá permissões que serão “ativadas” de acordo com o login do usuário. Por padrão haverá um administrador. 
    -A permissão de host: Será o usuário master do programa. Ele permite que o usuário adicione novos integrantes, 
    além de poder definir suas permissões. Além de poder editá-las a qualquer momento.  
    -A permissão de administrador: Poderá adicionar, excluir e editar receitas já criadas no sistema. 
    -A permissão de editor: Será responsável de editar receitas já criadas no sistema. 
    -A permissão do “cliente”: Tornará possível a busca de receitas em todo sistema. 
    -A permissão de visitantes (Usuário sem login): Apenas poderá ver o conteúdo da página. 
Haverá uma dashboard para usuários com permissão de host, administrador, editor e cliente. 
    -Host: Terá um botão para adicionar, excluir, editar usuário. Além de ter controle completo das receitas do sistema 
    -Administrador: Terá botões para adicionar excluir e editar receitas. 
    -Editor: Apenas os botões de editar receitas. 

Funcionalidades
-Cadastro de usuário
-Login
-Cadastro de receitas
-Edição de receitas
-Remoção de receitas
-Visualização de lista de  receitas.
-Visualização de receita individual
-Contagem de visualizações de receita
-Internacionalização

Descrição das telas
-Tela de cadastro de usuário
    A tela de cadastro de usuário está presente na tela de login também porém oculta. Ela é acessada/exibida assim que o usuário clicar em “Cadastrar”. 
    Após isso, será exibido os campos de “Nome”, “e-mail” e “senha”. Ao clicar em cadastrar, o sistema irá verificar se aquele usuário já existe, e em caso verdadeiro, 
    ele informa e pede para utilizar outro usuário. Em caso negativo de existência, o cadastro é realizado. 
    Todo usuário cadastrado será dado permissão de visitante por padrão.
-Tela de login
    A tela inicial em caso de usuário novo ou ainda não logado. Nesta tela estará presente a opção de login, assim como, de maneira oculta mas na mesma página, 
    a opção de cadastro. O login consiste no preenchimento dos campos de usuário e senha. O sistema irá validar as informações e permitir a entrada do usuário ou 
    recusar em caso negativo. Existe ainda a opção de recuperação de senha. Esta funcionalidade apenas existirá no front-end do código mas não enviará nenhum e-mail 
    para realizar a recuperação.
-Tela de visualização de receitas
    Essa tela apresentará uma lista de todas as receitas cadastradas no sistema, permitindo  que o usuário selecione uma das receitas para visualizá-la. 
    Caso o usuário logado tenha permissão de host ou de administrador, nesta página também estarão visíveis botões para que ele possa adicionar, 
    excluir ou editar receitas, sendo que, caso o usuário tenha permissão de editor, essa última opção também está disponível mas não as demais.
-Tela de cadastro de receita
    Esta tela conterá um formulário onde poderão ser inseridas as informações da receita. Os campos são os seguintes: nome,  tempo de preparo, rendimento (porções), 
    nível de dificuldade, lista de ingredientes, modo de preparo e o autor da receita. 
-Tela de edição de receita
    Esta tela irá conter as informações da receita selecionada, com a possibilidade de edição nos campos: nome,  tempo de preparo, rendimento (porções), 
    nível de dificuldade, lista de ingredientes, modo de preparo e o autor da receita, sendo identificada no banco pelo seu ID. 
-Tela de visualização individual de receita
    Nesta tela as informações cadastradas de uma receita serão apresentadas de forma organizada. Além disso, caso o usuário logado tenha permissão de administrador ou host, 
    será possível visualizar quem adicionou a receita e, caso ela tenha sido editada, quem editou..
